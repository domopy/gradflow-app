# 参与贡献

感谢你帮助改进研程。

## 开始之前

- 阅读`PROJECT_CONTEXT.md`了解产品边界；
- 阅读`AGENTS.md`了解代码规范；
- 搜索现有Issue，避免重复工作；
- 较大的功能或架构改动请先创建Issue讨论。

## 本地开发

```bash
npm install
npm run verify
```

Node.js最低版本为22.13，推荐使用Node.js 24 LTS。

## 分支与提交

- 从最新主分支创建短期功能分支；
- 一个Pull Request只解决一个清晰问题；
- 提交信息使用简洁的祈使句；
- 不提交构建产物、本地数据库、真实截图、聊天记录或API Key。

## 代码要求

- 遵循现有目录、命名和架构边界；
- 页面不直接执行SQL；
- AI输出必须经过运行时Schema校验；
- AI候选项未经用户确认不得写入正式日程；
- 业务逻辑改动必须添加或更新测试；
- 公共接口、配置、权限或数据格式变更必须同步更新文档。

## Pull Request检查清单

- [ ] 改动范围与Issue一致；
- [ ] 没有真实个人数据或密钥；
- [ ] 已运行`npm run verify`；
- [ ] 相关测试已添加或更新；
- [ ] 用户可见行为和文档保持一致；
- [ ] 涉及权限、隐私或数据删除时已说明风险。

## 报告问题

普通缺陷请使用Issue模板。安全问题请遵循`SECURITY.md`，不要公开披露敏感细节。
