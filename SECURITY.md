# 安全策略

## 支持范围

项目仍处于早期开发阶段，仅最新主分支和最新发布版本接受安全修复。

## 私密报告

请勿在公开Issue中提交以下内容：

- API Key、令牌或账号信息；
- 真实聊天记录或截图；
- 可识别个人身份的数据；
- 可直接利用的漏洞细节。

优先使用GitHub仓库的Private vulnerability reporting功能。若仓库尚未启用该功能，请通过维护者公开资料中提供的私密联系方式报告。

报告中请包含：

- 受影响版本和平台；
- 可复现步骤；
- 影响范围；
- 最小化的示例数据；
- 已知缓解方式。

维护者会在确认报告后协调披露时间。请在修复发布前避免公开漏洞细节。

## 安全边界

- DeepSeek API Key和EasyOCR Access Key存入SecureStore，不进入SQLite、日志或导出；
- AI输出必须经过Zod校验和人工确认；
- 应用不自动读取聊天或整个相册；
- 删除事项时同步取消关联通知；
- 测试和示例必须使用虚构数据。
