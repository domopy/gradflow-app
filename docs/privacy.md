# 隐私说明

研程采用本地优先的数据策略。

## 本地处理的数据

- 用户手动输入的标题、时间、地点、要求和提交方式；
- 本地通知标识；
- 用户主动写入系统日历的事项。

这些结构化数据默认保存在当前设备的应用数据库中。应用不要求注册，不使用云数据库，也不自动读取聊天记录或相册。

## DeepSeek文字识别

- 仅在用户主动点击“开始识别”后发送所粘贴的文字；
- 请求发送到用户在设置中配置的DeepSeek或兼容服务；
- API Key保存在SecureStore，不写入SQLite、日志或导出文件；
- 模型结果必须通过运行时Schema校验并由用户确认；
- 未经确认的候选项不会写入正式日程或创建提醒。

## 图片与EasyOCR

- 应用不会自动读取相册，只处理用户主动选择或拍摄的图片；
- 用户确认后，压缩图片会上传到所配置的EasyOCR服务；
- EasyOCR返回的识别文字随后发送到所配置的DeepSeek兼容服务；
- EasyOCR Access Key保存在SecureStore，不写入SQLite、日志或导出文件；
- 默认在确认结束后删除压缩临时图片；
- 用户可选择在应用私有目录保留压缩副本，用于之后核对来源；
- 删除某个来源关联的最后一项日程时，对应压缩副本会一并删除。

## 权限

- 通知权限：仅在用户为事项选择提醒后请求；
- 日历权限：仅在用户主动选择写入系统日历后请求。
- 相册权限：使用系统选择器读取用户主动选择的图片；
- 相机权限：仅在用户主动选择拍摄通知时请求。

权限被拒绝不会阻止用户保存普通本地事项。

## JSON备份

- 备份包含结构化日程、证据文字和变更历史；
- 备份不包含API Key、系统通知标识和来源图片；
- 导出后由用户选择分享目标，项目不提供自建云端；
- 恢复会替换当前本地数据，并清理旧提醒和来源图片；
- 恢复JSON会在修改数据库前完成格式和版本校验。

DeepSeek API Key和EasyOCR Access Key应保存在SecureStore中，不得写入SQLite、日志、仓库或数据导出。
